# 작업 워크플로우

각 단계는 전문 에이전트가 담당하며, 비전(오케스트레이터)이 단계 간 흐름을 조율한다.

```
요구사항 분석 및 설계          [planner]
  → PRD 문서 작성              [prd-generator]
  → PRD 문서 검증              [prd-validator]
    → 검증 실패 시: PRD 문서 수정 (90점 이상이 될 때까지 반복 및 승인 요청)
  → 아키텍처 설계              [architect]
  → 개발 계획 수립 (로드맵)    [planner]
  → Task 작성                  [비전 → pm]
  → TDD 방법론 코드 구현       [frontend / backend / app-developer / devops]
  → 코드 검증                  [code-reviewer]
    → 검증 실패 시: 코드 수정 후 재검증 반복
  → 개발 완료
  → 테스트 케이스 작성         [qa]
  → 단위 테스트                [qa]
  → 통합 테스트                [qa]
    → 테스트 실패 시: 코드 수정부터 반복
  → E2E 테스트 자동화          [qa / frontend]
  → 기술 문서 작성             [doc-writer]
  → PM 보고서 생성             [pm] ← 강제
  → 비전 검수 및 대시보드 생성 요청 [비전 → pm] ← 강제
```

---

## PM 보고서 강제 규칙

모든 개발 사이클 종료 시 **PM 에이전트는 반드시 두 가지 형식의 보고서를 생성**해야 한다.

### 1. HTML 보고서 (`reports/project-status.html`)

사용자·관리자용 시각적 대시보드. 아래 섹션을 모두 포함해야 한다.

| 섹션 | 내용 |
|------|------|
| 프로젝트 개요 | 목표, 현재 단계, 전체 진행률 (%) |
| 로드맵 | 마일스톤별 타임라인 |
| 에이전트별 현황 | 각 서브에이전트 작업 상태 카드 |
| 진행 중인 작업 | 현재 실행 중인 태스크 목록 |
| 완료 목록 | 완료된 태스크 및 산출물 |
| 블로킹 이슈 | 해결 필요 항목 (우선순위) |
| 다음 작업 | 다음 단계 우선순위 목록 |

### 2. MD 보고서 (`reports/project-status.md`)

비전(오케스트레이터)용 텍스트 보고서. 아래 섹션을 포함한다.

| 섹션 | 내용 |
|------|------|
| 요약 | 한 줄 현황 및 전체 진행률 |
| 완료된 작업 | 에이전트별 완료 항목 |
| 진행 중 | 현재 실행 중인 작업 및 담당 에이전트 |
| 블로킹 이슈 | 에스컬레이션 필요 항목 |
| 다음 단계 | 우선순위 기반 다음 작업 목록 |
| 리스크 | 일정·품질·기술 리스크 |

## 비전 검수 및 대시보드 생성 흐름 (강제)

```
PM → 보고서 2종 생성 완료 → 비전에게 보고
  → 비전: HTML 보고서 누락 섹션 확인
    → 누락 있음: PM에게 보완 요청 → 재생성
    → 누락 없음: PM에게 대시보드 페이지 생성 요청
      → PM → 대시보드 생성 [dashboard/index.html]
        → 비전에게 최종 보고
```

## 문서 관리 강제 규칙

**모든 MD 문서는 반드시 `doc/` 폴더 아래 분류별 하위 폴더에 저장한다. 루트 또는 임의 경로에 문서 생성 금지.**

| 폴더 | 저장 문서 | 담당 에이전트 |
|------|-----------|---------------|
| `doc/planning/` | 페르소나, 마일스톤, PRD, WBS, FSD, MVP | planner, prd-generator, prd-validator |
| `doc/architecture/` | C4 모델, ERD, API 계약, ADR | architect |
| `doc/development/` | 기술 스택, 코딩 컨벤션, 환경 설정 | backend, frontend, app-developer |
| `doc/api/` | API 엔드포인트 명세, 요청/응답 예시 | doc-writer, backend |
| `doc/test/` | 테스트 케이스, QA 보고서 | qa |
| `doc/reports/` | PM 보고서 (project-status.md, 마일스톤 보고서) | pm |
| `doc/changelog/` | 버전별 변경 이력 | doc-writer |
| `doc/guides/` | 개발자 가이드, 배포 가이드, 사용자 매뉴얼 | doc-writer, devops |

> PM HTML 보고서(`reports/project-status.html`)와 대시보드(`dashboard/index.html`)는 별도 폴더에 유지한다.

## 대시보드 필수 구성 요소 (`dashboard/index.html`)

PM 에이전트가 생성하는 대시보드는 아래 항목을 모두 포함해야 한다.

### 시스템 현황 패널

| 항목 | 내용 |
|------|------|
| 서브에이전트 | 전체 에이전트 목록, 역할, 현재 상태 |
| Skills | 등록된 스킬 목록 및 설명 |
| Hooks | 설정된 훅 목록 및 트리거 조건 |
| Rules | 적용 중인 규칙 파일 목록 및 요약 |

### 프로젝트 관리 패널

| 항목 | 내용 |
|------|------|
| PRD | 제품 요구사항 문서 링크 및 승인 상태 |
| 로드맵 | 마일스톤별 진행 타임라인 (간트 차트) |
| 작업 진행 상태 | 칸반보드 (대기 / 진행 중 / 완료 / 블로킹) |
| 완료된 작업 | 완료 태스크 목록 및 산출물 링크 |
| 진행 중인 작업 | 현재 작업 중인 태스크 및 담당 에이전트 |
| 다음 개발 | 우선순위 기반 예정 작업 목록 |
