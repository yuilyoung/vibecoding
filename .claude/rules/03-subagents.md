# 서브에이전트 규칙

- 오케스트레이터(비전)로부터 작업을 할당 받아 진행하며 진행 상태를 메일박스로 비동기 보고한다.
- 서브에이전트들 간에 메일박스로 작업 상태를 주고 받으며 자체적으로 토론 및 수정한다.

## 메일박스 프로토콜 (강제)

### 메일박스 경로

```
work/{프로젝트명}/.mailbox/
├── {발신자}-to-{수신자}.md    — 1:1 메시지
├── broadcast.md               — 전체 공지 (비전/PM → 전원)
└── escalation.md              — 에스컬레이션 (PM → 비전)
```

### 메시지 형식

```markdown
## [{타임스탬프}] {발신자} → {수신자}

**상태**: request | response | escalation | complete
**내용**: {작업 내용 또는 결과}
**다음 단계**: {필요한 후속 작업}
```

### 통신 규칙

| 규칙 | 설명 |
|------|------|
| 직접 통신 금지 | 에이전트 간 직접 호출 불가, 반드시 메일박스 경유 |
| PM 경유 원칙 | 팀 간 작업 요청은 PM을 통해 조율 |
| 에스컬레이션 | PM이 해결 불가 → escalation.md → 비전 판단 |
| 완료 보고 필수 | 모든 작업 완료 시 PM에게 complete 메시지 |

### 충돌 해결

```
에이전트 A ↔ 에이전트 B 의견 충돌
  → PM 중재 (메일박스에서 양측 의견 수집)
    → 해결 가능: PM이 결정 후 양측에 통보
    → 해결 불가: PM → escalation.md → 비전 판단
```
