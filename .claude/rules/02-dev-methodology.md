# 개발 방법론 강제 규칙

하나의 개발을 진행하기 전 아래 방법론을 반드시 수행한다:

| 단계 | 산출물 | 승인 |
|------|--------|------|
| 페르소나 | 사용자/이해관계자 정의 | 자동 |
| 마일스톤 | 단계별 목표 및 일정 | 자동 |
| PRD | 제품 요구사항 문서 | 비전 승인 필요 |
| WBS | 작업 분류 체계 | 자동 |
| FSD | 기능 상세 설계서 | 자동 |
| MVP | 최소 기능 제품 범위 정의 | 자동 |
| TDD | 테스트 주도 개발 | 자동 |
| 칸반보드 | 작업 상태 시각화 | 자동 |

## 분석 단계 자동 진행 (강제)

- 분석 작업(Read 권한만 필요한 단계)은 사용자 승인 없이 즉시 진행한다.
- 파일 읽기, 검색, 코드 분석, 구조 파악 등 읽기 전용 작업은 자동 수행.
- 사용자 승인이 필요한 시점: PRD 승인, 아키텍처 승인, 코드 리뷰 승인, 배포 승인.

| 자동 진행 (승인 불필요) | 승인 필요 |
|------------------------|-----------|
| 요구사항 분석 | PRD 최종 승인 |
| 기존 코드/구조 분석 | 아키텍처 최종 승인 |
| 페르소나/마일스톤 정의 | 코드 리뷰 최종 승인 |
| WBS/FSD/MVP 작성 | 배포 승인 |
| 테스트 케이스 분석 | 보안 Critical 이슈 |
