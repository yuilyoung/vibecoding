# WBS — 작업 분류 체계

프로젝트: Unity 기반 4v4 2D FPS 게임
작성일: 2026-04-05
작성자: planner 에이전트

---

## 1. WBS 개요

본 문서는 프로젝트 전체 작업을 계층적으로 분류한 작업 분류 체계이다.
각 태스크는 담당 에이전트, 예상 소요 시간, 선행 의존성을 명시한다.

**에이전트 코드**
- PLAN: planner (기획자)
- ARCH: architect (아키텍처)
- FE: frontend (프론트엔드)
- BE: backend (백엔드)
- APP: app-developer (응용 개발 / Unity)
- QA: qa (품질 보증)
- OPS: devops (DevOps)
- DOC: doc-writer (문서 작성)

## 4. 의존성 핵심 경로 (Critical Path)

```
1.1.3 (PRD) → 1.1.7 (PRD 검증) → 1.3.5 (API 계약)
  → 4.1.2 (NetworkManager) → 4.2.1 (동기화)
  → 4.4.1 (멀티플레이 테스트) → 6.1.2 (E2E)
  → 6.1.6 (릴리즈 빌드)
```

핵심 경로 상 태스크 지연 시 전체 일정에 직접 영향을 미치므로 우선순위 관리가 필요하다.
