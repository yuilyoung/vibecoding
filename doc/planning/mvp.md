작성일: 2026-04-05

## 1. MVP 정의

MVP(Minimum Viable Product)는 핵심 가치를 검증하기 위한 최소 기능 제품이다.
본 프로젝트의 핵심 가치는 다음과 같다.

> **"팀 기반 전술 전투의 즐거움을 4명이 실시간으로 함께 경험할 수 있는가?"**

MVP는 이 핵심 가치를 검증하는 데 필요한 기능만 포함하고,
나머지 기능은 검증 이후 단계적으로 추가한다.

---

## 2. MVP 핵심 가치 가설

| 가설 ID | 가설 내용 | 검증 방법 |
|---------|-----------|-----------|
| H-001 | 2D 탑다운 FPS가 팀 전술 전투로서 충분히 재미있다 | 내부 플레이테스트 5회, 재플레이율 측정 |
| H-002 | 4v4 매칭이 원활하게 이루어진다 | 4명 동시 접속 테스트, 레이턴시 측정 |
| H-003 | 라운드 기반 구조가 적절한 긴장감과 리듬을 제공한다 | 플레이어 세션 시간 및 피드백 수집 |

## 3. MVP 포함 기능 (In MVP)

### 3.1 게임플레이 핵심 기능

| 기능 | 구현 범위 | 이유 |
|------|-----------|------|
| 캐릭터 이동 (8방향, 마우스 조준) | 완전 구현 | 게임의 가장 기본 조작, 없으면 게임 불가 |
| 무기 시스템 — 소총 1종만 | 단일 무기 | 전투 핵심 검증에 1종으로 충분 |
| HP 시스템 및 사망 처리 | 완전 구현 | 전투 의미를 부여하는 핵심 요소 |
| 재장전 시스템 | 기본 구현 | 전투 리듬 형성에 필요 |
| 팀 시스템 (Red/Blue 2팀) | 완전 구현 | 4v4 팀 전투의 전제 조건 |
| 라운드 시스템 (Best of 5) | 완전 구현 | 승패 구조 없으면 게임 방향성 없음 |

### 3.2 멀티플레이 기능

| 기능 | 구현 범위 | 이유 |
|------|-----------|------|
| 로컬 네트워크(LAN) 4v4 멀티플레이 | 완전 구현 | 핵심 가치 "함께 경험"의 필수 조건 |
| 플레이어 상태 동기화 (위치, HP, 사망) | 완전 구현 | 멀티플레이 최소 요건 |
| 방 생성 및 참가 (호스트-클라이언트) | 기본 구현 | 플레이어가 같은 게임에 들어오는 수단 |

### 3.3 맵 및 환경

| 기능 | 구현 범위 | 이유 |
|------|-----------|------|
| 기본 맵 1종 (장애물 10개 이상) | 완전 구현 | 전술 전투를 위한 공간 필수 |
| 팀별 스폰 지점 | 완전 구현 | 라운드 시작 조건 |
| 맵 경계 처리 | 완전 구현 | 게임 진행 안정성 |

### 3.4 UI / 정보 표시

| 기능 | 구현 범위 | 이유 |
|------|-----------|------|
| 기본 HUD (HP 바, 탄약, 팀 스코어) | 완전 구현 | 플레이어가 게임 상태를 파악하는 필수 정보 |
| 라운드 결과 팝업 | 간소화 구현 | 라운드 진행 확인 |
| 최종 결과 화면 | 간소화 구현 | 매치 종료 인식 |
| 방 대기실 UI | 기본 구현 | 게임 시작 진입점 |

## 4. MVP 제외 기능 (Out of MVP)

| 기능 | 제외 이유 | 추가 예정 단계 |
|------|-----------|----------------|
| 무기 2·3종 (산탄총, 저격총) | 1종으로 핵심 전투 검증 가능 | M4 |
| 맵 2종 | 1종으로 레이아웃 검증 가능 | M4 |
| 사운드 이펙트 | 게임성 검증에 필수 아님 | M4 |
| 비주얼 이펙트 (VFX) | 게임성 검증에 필수 아님 | M4 |
| 킬 피드 UI | HUD에서 제외, 콘솔 로그로 대체 | M4 |
| 캐릭터 커스터마이징 | 핵심 가치와 무관 | 출시 후 검토 |
| 랭크/매칭 시스템 | 플레이어 기반 구축 후 필요 | 출시 후 검토 |
| 인터넷 멀티플레이 (Relay) | LAN으로 핵심 가치 검증 가능 | M3 |
| 음성 채팅 | 텍스트 채팅 또는 외부 도구 사용 | 출시 후 검토 |
| 관전 모드 | 핵심 가치와 무관 | 출시 후 검토 |
| 리플레이 시스템 | 핵심 가치와 무관 | 출시 후 검토 |
| 메인 메뉴 (완성형) | 기능 진입용 임시 UI로 대체 | M4 |
| 설정 화면 | 하드코딩으로 임시 처리 | M4 |
| 헤드샷 판정 차등 | 일반 피해 판정으로 단순화 | M2 이후 |

## 5. MVP 성공 기준

| 지표 | 목표값 | 측정 방법 |
|------|--------|-----------|
| 4명 동시 접속 안정성 | 오류 없이 1매치 완주 | 내부 테스트 |
| 네트워크 레이턴시 | LAN 환경에서 50ms 이하 | Unity Stats 측정 |
| 라운드 시스템 정확도 | 승패 판정 오류 0건 | 자동화 테스트 |
| 평균 세션 시간 | 15분 이상 | 내부 플레이테스트 측정 |
| 재플레이 의향 | 테스터 5명 중 3명 이상 "또 하고 싶다" | 설문 |

## 6. MVP 완료 정의 (Definition of Done)

아래 항목 전체 충족 시 MVP 완료로 판정한다.

- [ ] 4명의 플레이어가 LAN 환경에서 동시 접속하여 게임 시작 가능
- [ ] Red Team / Blue Team 2팀 구성, 팀별 4명 배치
- [ ] 소총으로 사격 가능, 피격 시 HP 감소, HP 0 시 사망 처리
- [ ] 한 팀 전멸 시 라운드 종료, 점수 +1
- [ ] Best of 5 기준 3라운드 먼저 획득 팀 승리 → 결과 화면 표시
- [ ] HUD에 HP, 탄약, 팀 스코어 실시간 표시
- [ ] 모든 플레이어 상태가 4개 클라이언트에 동기화
- [ ] 단위 테스트 커버리지 80% 이상 (이동, 전투, 라운드 모듈)
- [ ] 통합 테스트: 라운드 시작~종료 플로우 자동화 통과

## 7. MVP 이후 로드맵 (Post-MVP)

```
MVP 완료 (M2)
    │
    ▼
M3 — 인터넷 멀티플레이 (Relay 서버)
    - 랜선 없이 인터넷으로 게임 가능
    - 자동 매칭 기능
    │
    ▼
M4 — 콘텐츠 확장
    - 무기 2종 추가 (산탄총, 저격총)
    - 맵 1종 추가
    - 사운드, VFX, 완성형 UI
    │
    ▼
M5 — 출시 준비
    - E2E 테스트, 성능 최적화
    - Windows 빌드 배포
    │
    ▼
출시 후 검토
    - 랭크 시스템
    - 캐릭터 커스터마이징
    - 관전 / 리플레이 모드
    - 모바일 플랫폼 포팅
```
