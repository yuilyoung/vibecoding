## 2. PRD 작성 과정의 가정 (Assumptions)

초안에 명시되지 않았으나 PRD 작성 시 적용한 가정 목록이다.
추후 실제 개발 착수 전 아키텍처 설계 단계에서 재확인 권장.

| ID | 가정 내용 | 적용 위치 | 확인 권장 단계 |
|----|-----------|-----------|---------------|
| A-001 | UGS Relay는 WebGL 빌드에서 WebSocket 기반으로 정상 동작한다고 가정 | FR-009, FR-015 | M3 착수 전 PoC |
| A-002 | NGO(Netcode for GameObjects)는 WebGL 빌드를 공식 지원한다고 가정 | FR-008, 기술 스택 | M0 환경 세팅 시 확인 |
| A-003 | 소총(Rifle)의 발사 방식은 자동(클릭 유지 시 연속 발사)을 기본으로 적용 | FR-002 | M1 프로토타입 플레이테스트 후 조정 |
| A-004 | 헤드샷 판정은 MVP에서는 단순화(일반 피해)하고 M2 이후 차등 적용 | FR-003 | M2 착수 전 재확인 |
| A-005 | 매칭 알고리즘 중 레벨·랭크 기반은 M3 이후 구현이며 MVP에서는 랜덤만 구현 | FR-007 | M3 착수 전 재확인 |
| A-006 | WebGL 빌드의 최소 브라우저 사양은 Chrome 최신, Firefox 최신, Edge 최신으로 한정 | NFR-017 | M5 QA 단계에서 브라우저 호환성 테스트 |
| A-007 | UGS 무료 플랜 내에서 동시 접속 8명(4v4) 처리가 가능하다고 가정 | FR-009 | M0 사전 준비 시 UGS 요금제 확인 |
| A-008 | 자체 서버 전환은 정식 출시(M5) 이후이며 현 PRD 범위에서 설계 세부사항은 제외 | 섹션 9.1 | M5 이후 별도 PRD 작성 |

## 4. prd-validator 검증 요청

prd-generator → prd-validator

PRD v1.0 문서(`doc/planning/prd.md`) 작성이 완료되었습니다.
아래 기준에 따라 검증 및 점수화를 요청합니다.

- 검증 기준: 100점 기준 90점 이상 승인
- 검증 항목: 요구사항 완결성, 측정 가능성, 우선순위 명시, 기술 실현 가능성, 페르소나 연계성
- 90점 미만 시: 반려 항목과 수정 지시를 prd-generator에게 메일박스로 전달
- 90점 이상 시: 비전에게 승인 요청 후 architect, frontend, backend, qa에게 배포
