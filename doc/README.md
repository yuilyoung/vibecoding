# 문서 디렉토리

모든 프로젝트 문서는 이 폴더 아래 분류별로 관리된다.

## 폴더 구조

```
doc/
├── planning/       — 기획 문서 (페르소나, 마일스톤, PRD, WBS, FSD, MVP)
├── architecture/   — 아키텍처 문서 (C4 모델, ERD, API 계약, ADR)
├── development/    — 개발 문서 (기술 스택, 코딩 컨벤션, 환경 설정)
├── api/            — API 문서 (엔드포인트 명세, 요청/응답 예시)
├── test/           — 테스트 문서 (테스트 케이스, QA 보고서)
├── reports/        — PM 보고서 (project-status.md, 마일스톤 보고서)
├── changelog/      — 변경 이력 (버전별 변경 사항)
└── guides/         — 가이드 (개발자 가이드, 배포 가이드, 사용자 매뉴얼)
```
