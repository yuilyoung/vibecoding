# 페르소나 (Persona)

프로젝트: Unity 기반 4v4 2D FPS 게임
작성일: 2026-04-05
작성자: planner 에이전트

---

## 1. 페르소나 개요

본 문서는 Unity 기반 4v4 2D FPS 게임의 주요 사용자 및 이해관계자를 정의한다.
대상 플레이어층과 개발팀 이해관계자를 구분하여 각 그룹의 목표, 고통점, 행동 패턴을 기술한다.

---

## 2. 주요 사용자 페르소나

### 페르소나 A — 코어 게이머 "전략형 팀플레이어"

| 항목 | 내용 |
|------|------|
| 이름 | 박준혁 |
| 나이 | 22세 |
| 직업 | 대학생 (컴퓨터공학과) |
| 플레이 환경 | PC, 하루 2~4시간 게임 |
| 주 플레이 장르 | 팀 FPS (CS2, Valorant), 탑다운 슈터 |

**목표 (Goals)**
- 친구들과 함께 팀워크를 발휘한 전술 전투를 즐기고 싶다.
- 포지션별 역할 분담과 팀 전략으로 승리하는 경험을 원한다.
- 실력 향상에 따른 명확한 피드백과 랭크 성장을 원한다.

**고통점 (Pain Points)**
- 팀원 없이 솔로 플레이 시 전략성이 무너지는 게임에 실망한다.
- 스킬 격차가 커서 신규 유저와 기존 유저가 함께 플레이하기 어렵다.
- 접속 대기 시간이 길거나 매칭 품질이 낮으면 이탈한다.

**행동 패턴**
- 디스코드로 팀을 구성하고 전략을 논의한 뒤 게임에 접속한다.
- 킬/데스 비율, 팀 기여도 지표를 확인하며 플레이를 분석한다.
- 신규 맵·무기·스킨 업데이트 시 커뮤니티에 적극 참여한다.

### 페르소나 B — 캐주얼 게이머 "가볍게 즐기는 플레이어"

| 항목 | 내용 |
|------|------|
| 이름 | 이수진 |
| 나이 | 19세 |
| 직업 | 고등학생 / 아르바이트 |
| 플레이 환경 | PC 또는 노트북, 하루 1시간 이내 |
| 주 플레이 장르 | 가볍게 즐기는 아케이드, 모바일 슈터 |

**목표 (Goals)**
- 복잡한 조작 없이 빠르게 게임에 합류하여 즐겁게 플레이하고 싶다.
- 친구와 함께 플레이하며 채팅과 소통을 즐기고 싶다.
- 짧은 게임 시간 안에도 성취감을 느끼고 싶다.

**고통점 (Pain Points)**
- 조작이 복잡하거나 학습 곡선이 가파르면 흥미를 잃는다.
- 게임 설치 과정이 번거롭거나 사양이 높으면 포기한다.
- 팀원이 극단적으로 경쟁적이면 불쾌한 경험을 한다.

**행동 패턴**
- 유튜브/트위치에서 게임 플레이 영상을 보고 진입을 결정한다.
- 친구 초대 기능이나 파티 시스템으로 합류한다.
- 짧은 세션 후 결과를 SNS에 공유하기도 한다.

### 페르소나 C — 인디 게임 콘텐츠 크리에이터

| 항목 | 내용 |
|------|------|
| 이름 | 김민준 |
| 나이 | 27세 |
| 직업 | 유튜버 / 스트리머 |
| 플레이 환경 | 고사양 PC, 방송 장비 보유 |
| 주 플레이 장르 | 인디 게임, 멀티플레이 게임 전반 |

**목표 (Goals)**
- 시청자에게 흥미로운 게임플레이를 보여줄 수 있는 독특한 비주얼과 액션을 원한다.
- 게임 내 인상적인 순간(클립)을 쉽게 포착할 수 있는 게임을 선호한다.
- 제작사와 협업하여 게임을 홍보할 기회를 모색한다.

**고통점 (Pain Points)**
- 방송용 최적화가 안 되어 있으면 (프레임 드롭, 크래시) 방송 중단 위험이 있다.
- 스트리밍 중 스포일러성 요소나 게임 반복성이 높으면 시청자 이탈이 발생한다.
- 커뮤니티 관리 도구나 제작사 소통 창구가 부족하면 협업이 어렵다.

**행동 패턴**
- 얼리 액세스 또는 베타 테스트에 적극 참여한다.
- 게임의 독특한 메카닉이나 하이라이트 클립을 편집해 업로드한다.
- 팔로워들과 함께 팀 구성하여 직접 플레이한다.

## 3. 이해관계자 페르소나

### 이해관계자 A — 게임 개발팀 리드

| 항목 | 내용 |
|------|------|
| 이름 | 최성민 |
| 역할 | 프로젝트 리드 / 오케스트레이터(비전) |
| 관심사 | 프로젝트 일정 준수, 품질 확보, 팀 생산성 |

**목표**
- MVP를 기한 내에 출시하여 시장 반응을 빠르게 검증한다.
- 팀원 간 명확한 역할 분담으로 중복 작업과 병목을 최소화한다.
- 코드 품질과 테스트 커버리지를 유지하여 기술 부채를 억제한다.

**고통점**
- 요구사항 변경이 잦을 때 전체 일정이 흔들린다.
- 에이전트 간 커뮤니케이션 오버헤드로 생산성이 저하될 수 있다.

---

### 이해관계자 B — QA 엔지니어

| 항목 | 내용 |
|------|------|
| 이름 | 정다은 |
| 역할 | 품질 보증 (QA) |
| 관심사 | 버그 사전 발견, 테스트 자동화, 출시 안정성 |

**목표**
- 멀티플레이 환경에서 발생하는 동기화 버그와 네트워크 이슈를 사전에 발견한다.
- 테스트 케이스를 체계화하여 반복 테스트를 자동화한다.

**고통점**
- FSD가 불완전하면 테스트 케이스 작성이 어렵다.
- 멀티플레이 환경 재현이 어려워 간헐적 버그 검증에 시간이 많이 소요된다.

---

### 이해관계자 C — 백엔드 개발자

| 항목 | 내용 |
|------|------|
| 이름 | 한승우 |
| 역할 | 서버 및 네트워크 개발 |
| 관심사 | 서버 안정성, 매칭 시스템, 실시간 동기화 |

**목표**
- 4v4 실시간 전투를 지연 없이 처리하는 서버 구조를 설계한다.
- 매칭 알고리즘으로 공정한 팀 대전을 제공한다.

**고통점**
- 네트워크 레이턴시 문제로 클라이언트-서버 동기화가 어긋나는 경우가 많다.
- PRD에 API 계약이 불명확하면 개발 후 재작업이 발생한다.

## 4. 핵심 사용자 여정 (User Journey)

```
[발견] 유튜브/SNS 영상 시청 또는 친구 추천
    ↓
[설치] 게임 다운로드 및 설치 (간단한 설치 과정)
    ↓
[온보딩] 튜토리얼 또는 트레이닝 룸에서 기본 조작 학습
    ↓
[매칭] 로비에서 팀 구성 또는 자동 매칭
    ↓
[플레이] 4v4 전술 전투 (라운드 기반)
    ↓
[결과] 스코어보드 확인, 개인/팀 기여도 피드백
    ↓
[재플레이] 다음 매치 참여 또는 로비 복귀
```
