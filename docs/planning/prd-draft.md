프로젝트: Unity 기반 4v4 2D FPS 게임
문서 버전: v0.1 (초안)
작성일: 2026-04-05
작성자: planner 에이전트
상태: 초안 (prd-validator 검증 전)

---

## 1. 배경 및 목적

### 1.1 배경
팀 기반 멀티플레이 FPS 장르는 전 세계적으로 높은 플레이어 수와 시장 규모를 유지하고 있다.
기존 3D FPS 게임과 달리, 2D 탑다운 방식은 낮은 하드웨어 요구사항과 직관적인 전술 시각화를 제공하여
캐주얼 및 코어 플레이어 모두에게 접근성이 높다.

본 프로젝트는 Unity 2D 환경에서 4v4 팀 전술 전투를 구현한 멀티플레이 게임을 개발한다.

### 1.2 목적
- 팀 기반 전술 전투 경험을 제공하는 2D 멀티플레이 FPS 게임 출시
- 캐주얼 플레이어와 코어 플레이어 모두가 즐길 수 있는 접근성 확보
- MVP 출시 후 플레이어 피드백을 통한 빠른 반복 개선

### 1.3 성공 지표
- 동시 접속자 4명이 지연 없이 게임 가능 (레이턴시 100ms 이하)
- 첫 플레이 후 재플레이율 60% 이상 (초기 테스트 그룹 기준)
- 평균 세션 시간 15분 이상

## 3. 사용자 스토리

### 3.1 핵심 사용자 스토리

**ST-001** — 매칭 참가
> 플레이어로서, 로비에서 팀을 구성하거나 자동 매칭을 통해 4v4 게임에 참가할 수 있어야 한다.
> 매칭 대기 시간이 30초를 초과하면 현재 상태를 안내받고 싶다.

**ST-002** — 캐릭터 이동
> 플레이어로서, 키보드(WASD)로 캐릭터를 8방향 이동시키고 마우스 방향을 바라볼 수 있어야 한다.

**ST-003** — 사격
> 플레이어로서, 마우스 좌클릭으로 현재 장착한 무기를 발사하고 적에게 피해를 줄 수 있어야 한다.

**ST-004** — 피격 및 사망
> 플레이어로서, 적의 탄환에 피격 시 HP가 감소하고 HP가 0이 되면 해당 라운드에서 사망 처리되어야 한다.

**ST-005** — 라운드 진행
> 플레이어로서, 한 팀이 전멸하면 라운드가 종료되고 점수가 올라가는 것을 HUD에서 실시간으로 확인할 수 있어야 한다.

**ST-006** — 게임 종료
> 플레이어로서, 설정된 라운드 수를 먼저 달성한 팀이 승리하고 최종 결과 화면을 통해 팀별 통계를 확인할 수 있어야 한다.

**ST-007** — 무기 교체
> 플레이어로서, 게임 중 숫자키 또는 마우스 휠로 보유 무기를 교체할 수 있어야 한다.

**ST-008** — 재장전
> 플레이어로서, R 키를 눌러 재장전하거나 탄창이 비었을 때 자동으로 재장전이 시작되어야 한다.

## 4. 기능 요구사항

### 4.1 게임플레이

**FR-001 캐릭터 이동**
- WASD 키로 8방향 이동 (대각선 포함)
- 이동 속도: 기본 5 units/sec (무기에 따라 최대 20% 감소 가능)
- 장애물 충돌 시 이동 불가
- 마우스 커서 방향으로 캐릭터 회전

**FR-002 무기 시스템**
- 무기 3종: 소총(Rifle), 산탄총(Shotgun), 저격총(Sniper)
- 각 무기별 스탯: 데미지, 연사속도(RPM), 탄창 크기, 재장전 시간, 사거리
- 탄환 물리: 발사 속도, 관통 여부, 산탄 패턴

| 무기 | 데미지 | RPM | 탄창 | 재장전(초) | 사거리 |
|------|--------|-----|------|-----------|--------|
| 소총 | 25 | 600 | 30 | 2.0 | 무제한 |
| 산탄총 | 15x6 | 80 | 6 | 2.5 | 중거리 |
| 저격총 | 100 | 40 | 5 | 3.0 | 무제한 |

**FR-003 HP 시스템**
- 기본 HP: 100
- 피격 시 데미지 계산 적용
- 헤드샷 영역: 데미지 1.5배
- HP 0 이하 시 사망
- 라운드 시작 시 HP 전체 회복

**FR-004 라운드 시스템**
- 기본 라운드 수: 5라운드 (홀수 설정 권장)
- 라운드 종료 조건: 한 팀 전원 사망
- 라운드 시작 카운트다운: 5초
- 라운드 종료 후 다음 라운드 대기: 3초
- 먼저 3라운드 획득 팀이 최종 승리 (Best of 5 기준)

**FR-005 팀 시스템**
- Red Team / Blue Team 2팀 구성
- 팀당 최대 4명
- 팀 색상으로 아군/적군 구분
- 아군 피해 없음 (팀킬 불가)
- 팀별 스폰 구역 지정

### 4.2 네트워크 및 멀티플레이

**FR-006 방(Room) 시스템**
- 방 생성: 호스트가 방 이름, 최대 인원 설정
- 방 참가: 방 코드 또는 목록에서 선택
- 최대 8명(4v4) 수용
- 호스트 이전: 호스트 연결 끊김 시 자동 이전

**FR-007 게임 상태 동기화**
- 플레이어 위치, 회전각 동기화 (서버 권위)
- 탄환 발사 이벤트 동기화
- HP 변화 동기화
- 라운드 상태 동기화

**FR-008 레이턴시 허용 기준**
- 정상 게임플레이: 100ms 이하
- 경고 상태: 100~200ms (시각적 경고 표시)
- 강제 처리: 200ms 초과 지속 시 연결 불안정 메시지

### 4.3 맵 및 환경

**FR-009 맵 요구사항**
- 맵 크기: 화면 기준 20x20 단위 이상
- 장애물: 최소 10개 이상 배치 (엄폐물 역할)
- 맵 경계: 캐릭터 이탈 불가 처리
- 팀별 스폰 구역: 맵 양 끝에 분리 배치
- 맵 2종: 실내(박스형), 실외(오픈형)

### 4.4 UI / UX

**FR-010 HUD**
- HP 바: 현재 HP / 최대 HP 시각적 표시
- 탄약 표시: 현재 탄창 / 보유 탄약
- 팀 스코어: Red Team vs Blue Team 라운드 수
- 라운드 타이머 (라운드 최대 시간 설정 시)
- 킬 피드: 우측 상단, 최근 킬 이벤트 3건 표시

**FR-011 로비 UI**
- 방 생성 / 방 참가 버튼
- 방 목록 화면 (방 이름, 현재 인원/최대 인원, 상태)
- 팀 선택 화면 (Red / Blue)
- 준비 완료 버튼 (전원 준비 완료 시 호스트 게임 시작 가능)

**FR-012 결과 화면**
- 승리/패배 명확한 표시
- 팀별 총 라운드 수
- 개인 스탯: 킬, 데스, 어시스트
- 다음 게임 또는 로비 복귀 버튼

## 5. 비기능 요구사항

### 5.1 성능

**NFR-001** — 프레임 레이트
- 목표 FPS: 60 이상 (권장 사양 기준)
- 최소 FPS: 30 이상 (최소 사양 기준)
- 8명 동시 접속 시에도 FPS 저하 없음

**NFR-002** — 메모리 사용량
- 게임 플레이 중 RAM 사용량 2GB 이하
- 씬 전환 시 메모리 누수 없음

**NFR-003** — 네트워크 트래픽
- 클라이언트당 업스트림 32kbps 이하
- 패킷 손실 5% 이하 환경에서 정상 동작

### 5.2 보안

**NFR-004** — 서버 권위 처리
- 이동, 데미지 계산은 서버에서 처리 (클라이언트 치트 방지)
- 클라이언트의 직접 HP 조작 불가

**NFR-005** — 입력 검증
- 서버로 전달되는 모든 입력값 유효성 검증
- 비정상적인 이동 속도, 데미지 값 서버에서 거부

### 5.3 접근성

**NFR-006** — 시스템 요구사항
- 최소 사양: CPU i5 2세대, RAM 4GB, GPU 내장그래픽
- 권장 사양: CPU i5 8세대, RAM 8GB, GPU GTX 1060

**NFR-007** — 해상도 지원
- 지원 해상도: 1280x720, 1920x1080, 2560x1440
- 화면비 16:9 고정 (letterbox 처리)

### 5.4 유지보수성

**NFR-008** — 코드 품질
- 단위 테스트 커버리지 80% 이상
- 코드 리뷰 100점 기준 90점 이상 통과

## 7. 제약 사항

- Unity 2022 LTS 버전으로 고정 (호환성 및 LTS 지원 보장)
- 무료/오픈소스 에셋 우선 사용 (상업용 에셋은 별도 승인 필요)
- 멀티플레이는 LAN 및 인터넷 환경 모두 지원 (Relay 서버 검토)
- 출시 플랫폼은 PC(Windows)로 한정 (macOS/Linux는 M4 이후 검토)

## 8. 용어 정의 (Glossary)

| 용어 | 정의 |
|------|------|
| 라운드 | 한 팀이 전멸할 때까지의 단위 전투 |
| 매치 | 여러 라운드로 구성된 하나의 게임 세션 |
| 스폰 | 라운드 시작 시 캐릭터가 배치되는 지점 |
| 서버 권위 | 게임 상태의 최종 판단을 서버가 담당하는 방식 |
| 킬 피드 | 화면에 표시되는 킬 이벤트 목록 |
| HUD | Heads-Up Display, 게임 중 화면에 표시되는 정보 패널 |
